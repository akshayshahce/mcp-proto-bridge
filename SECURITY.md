# Security Policy

## Supported Versions

Security fixes are provided for the latest release on `main`.

## Reporting a Vulnerability

Do not open public GitHub issues for security vulnerabilities.

Report privately via repository security advisories:

1. Open the repository Security tab.
2. Choose "Report a vulnerability".
3. Include reproduction details, impact, and affected versions.

If security advisories are unavailable in your environment, contact repository maintainers through your internal security intake process.

## Disclosure and Response Expectations

1. Initial triage target: 3 business days.
2. Severity assessment target: 5 business days.
3. Fix timeline depends on impact and exploitability.
4. Coordinated disclosure is preferred after patch availability.
